# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Moon Cycle, please report it responsibly.

### How to Report

1. **Do NOT** open a public issue
2. Email the maintainer directly (check package.json or GitHub profile)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- Acknowledgment within 48 hours
- Regular updates on the progress
- Credit in the security advisory (unless you prefer to remain anonymous)

## Scope

### In Scope
- Client-side security issues
- Local storage vulnerabilities
- XSS vulnerabilities
- Data privacy issues

### Out of Scope
- Issues in third-party dependencies (report to the dependency maintainers)
- Social engineering attacks
- Physical access attacks

## Security Features

Moon Cycle is designed with privacy and security in mind:
- **No Backend**: All data stays on your device
- **No Cloud Sync**: No data transmission to external servers
- **Local Storage Only**: Data stored in browser's local storage
- **No Tracking**: No analytics or third-party services
- **No Authentication**: No passwords or accounts to compromise

## Best Practices for Users

- Keep your browser updated
- Use HTTPS when accessing the app
- Be cautious when using shared computers
- Regularly export your data as backup
- Clear browser data when using public computers

## Updates

Security updates will be released as soon as possible after a vulnerability is confirmed.
